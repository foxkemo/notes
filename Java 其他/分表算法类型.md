在 ShardingSphere 5.x 中，分片算法（Sharding Algorithm）非常丰富。除了你正在使用的 `CLASS_BASED`（自定义类算法），主要可以分为以下几大类：

### 1. 自动分片算法 (Automatic Sharding Algorithms)
这些算法通常用于 `AUTO` 分片策略，配置简单，不需要自己写逻辑。

*   **MOD（取模分片）**：
    直接对分片键进行取模。
    ```yaml
    type: MOD
    props:
      sharding-count: 4 # 分片数量
    ```
*   **HASH_MOD（哈希取模）**：
    先对分片键取 Hash 值，再取模。适合字符串类型的分片键。
    ```yaml
    type: HASH_MOD
    props:
      sharding-count: 4
    ```
*   **VOLUME_RANGE（容量范围分片）**：
    按照数据量大小范围分片（如 0-100万一条，100-200万一条）。
*   **BOUNDARY_RANGE（边界范围分片）**：
    自定义具体的数值边界（如 1-1000, 1000-5000...）。

---

### 2. 标准分片算法 (Standard Sharding Algorithms) - 最常用
适用于 `STANDARD` 分片策略（单分片键）。

*   **INLINE（行内表达式算法）**：
    **这是开发中最常用的算法**。使用 Groovy 表达式，不需要写 Java 代码。
    ```yaml
    type: INLINE
    props:
      algorithm-expression: ds_${shop_number % 2}.t_order_${shop_number % 16}
      allow-range-query-with-inline-sharding: true # 是否允许范围查询
    ```
*   **INTERVAL（时间范围分片）**：
    专门用于按时间（天、月、年）分片，非常适合日志表。
    ```yaml
    type: INTERVAL
    props:
      datetime-pattern: "yyyy-MM-dd HH:mm:ss" # 时间格式
      datetime-lower: "2023-01-01 00:00:00" # 起始时间
      datetime-upper: "2024-01-01 00:00:00" # 结束时间
      sharding-suffix-pattern: "yyyyMM"      # 表名后缀格式
      datetime-interval-amount: 1            # 时间间隔
      datetime-interval-unit: "MONTHS"       # 间隔单位
    ```

---

### 3. 复合分片算法 (Complex Sharding Algorithms)
适用于 `COMPLEX` 分片策略（多个分片键）。

*   **COMPLEX_INLINE**：
    可以使用多个分片键的行内表达式。

---

### 4. Hint 分片算法 (Hint Sharding Algorithms)
适用于 `HINT` 分片策略（由程序手动指定分片，而不是根据 SQL 里的字段）。

*   **HINT_INLINE**：
    基于 Hint 值的行内表达式。

---

### 5. 其他特殊算法
*   **COSID（基于 CosId 框架）**：
    美团开源的分布式 ID 生成及分片框架。提供 `COSID_MOD`、`COSID_INTERVAL` 等算法，性能极高。
*   **NANOID**：
    基于 NanoID 的分片算法。

---

### 总结：你应该选哪种？

1.  **如果逻辑简单**（如单纯取模）：
    直接用 **`MOD`** 或 **`HASH_MOD`**。
2.  **如果不想写 Java 代码，但逻辑稍微复杂**：
    用 **`INLINE`**。它是目前市面上最主流的选择，直接在 YAML 里写表达式即可。
3.  **如果按时间分片**：
    必须用 **`INTERVAL`**，因为它能完美处理跨月、跨年查询。
4.  **如果逻辑非常特殊**（如你的库表映射逻辑）：
    才使用 **`CLASS_BASED`**。

**注意：** 只有 **`CLASS_BASED`** 需要你提供 `algorithm-class-name`，其他的内置算法只需要指定 `type` 和对应的 `props` 即可。
