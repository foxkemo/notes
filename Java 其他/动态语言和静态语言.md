简单来说，**动态语言**与**静态语言**的核心区别在于：**“什么时候”确定变量的类型**，以及**“是否需要”显式声明类型**。

---

## 1. 核心区别：编译期 vs 运行期

## 静态语言 (Static Typing)

在**编译阶段**（写完代码，运行之前）就必须确定每个变量的类型。如果类型不匹配，编译器会直接报错，程序无法运行。

- **代表语言**：C, C++, Java, Go, Rust。
    
- **特点**：必须先声明类型，如 `int count = 10;`。
    

## 动态语言 (Dynamic Typing)

在**运行阶段**才去确定变量的类型。变量本身没有死板的类型限制，它可以随时指向不同类型的数据。

- **代表语言**：Python, JavaScript, Ruby, PHP, Lua。
    
- **特点**：不需要声明类型，直接赋值，如 `count = 10`。
    

---

## 2. 详细对比

|**特性**|**静态语言 (如 Java/C++)**|**动态语言 (如 Python/JS)**|
|---|---|---|
|**类型检查时间**|**编译时** (Compile-time)|**运行时** (Runtime)|
|**代码书写**|较繁琐，需写明 `int`, `String` 等|简洁，直接 `var` 或 `local`|
|**性能**|**高**（编译器提前做了优化）|**低**（运行时需要解释类型）|
|**错误发现**|早期发现，IDE 报错即刻知晓|只有运行到那一行报错才知道|
|**灵活性**|较低，重构时类型约束多|极高，代码量少，适合快速开发|

---

## 3. 形象的比喻

- **静态语言就像“入职面试”**：
    
    你在进入公司（运行）之前，必须先证明你的职业（类型）。如果你是会计，你只能做账；如果你想改做程序员，必须经过严格的重新考核（类型转换）。
    
- **动态语言就像“特种兵”**：
    
    你先进入战场（运行）。手拿枪你就是战士，手拿听诊器你就是医生。你的身份取决于你当下手里拿的是什么（赋值内容）。

## 4. 为什么 Lua 是动态语言？

回到你之前的 Redis Lua 脚本问题：

在 Lua 中，你可以这样写：

Lua

```
local x = 10       -- x 现在是数字
x = "hello"        -- x 瞬间变成了字符串，没问题！
```

这种灵活性让你在处理 Redis 返回的数据时非常方便。例如，`redis.call('GET', 'key')` 可能返回一个字符串，也可能返回 `false`（代表不存在），在 Lua 这种动态语言里，一个 `local` 变量就能轻松搞定。

---

## 5. 常见误区：强类型 vs 弱类型

很多人把动态语言等同于弱类型，其实这是两个维度：

- **动态/静态**：关注**什么时候**检查类型。
    
- **强/弱类型**：关注**是否允许隐式转换**。
    
    - **Python** 是“动态强类型”：`1 + "1"` 会报错，因为它不允许数字和字符串乱搞。
        
    - **JS** 是“动态弱类型”：`1 + "1"` 结果是 `"11"`，它会自动帮你转类型。
        
    - **Lua** 也是“动态弱类型”：它会尝试把字符串转成数字进行计算。
