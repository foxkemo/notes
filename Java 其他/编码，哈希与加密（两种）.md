这几个概念很多人会混在一起，我帮你用**一套清晰的分类 + 对比 + 实战理解**彻底理顺👇

---

# 一、先给你一张总图（最重要🔥）

```text
数据处理方式分 4 类：

1️⃣ 编码（Encoding）      → 可逆
2️⃣ 哈希（Hash）         → 不可逆
3️⃣ 对称加密（Symmetric）→ 可逆（一个key）
4️⃣ 非对称加密（Asymmetric）→ 可逆（公钥+私钥）
```

---

# 二、编码（Encoding）

## 👉 本质

```text
数据格式转换（不是为了安全）
```

---

## 👉 是否可逆

```text
✅ 可逆
```

---

## 👉 常见编码

* Base64（最常见🔥）
* Hex（16进制）
* URL编码

---

## 👉 示例

```text
hello → aGVsbG8=
```

👉 可以直接解码回来

---

## 👉 用途

* 网络传输
* JSON / HTTP
* 图片、Token

---

## ❗总结

```text
编码 ≠ 加密（没有安全性）
```

---

# 三、哈希（Hash）

## 👉 本质

```text
生成“数据指纹”
```

---

## 👉 是否可逆

```text
❌ 不可逆
```

---

## 👉 常见算法

* MD5（已不安全）
* SHA-256（常用🔥）
* bcrypt（存密码🔥）

---

## 👉 特点

* 固定长度
* 输入变一点 → 输出完全变
* 不可解密

---

## 👉 用途

* 密码存储
* 数据校验

---

# 四、加密（Encryption）

👉 分两大类👇

---

# 五、对称加密（Symmetric）

## 👉 本质

```text
加密 = key
解密 = key（同一个）
```

---

## 👉 是否可逆

```text
✅ 可逆
```

---

## 👉 常见算法

* AES（主流🔥）
* DES（淘汰）

---

## 👉 特点

✅ 快
❌ 密钥分发困难

---

## 👉 用途

* 文件加密
* 数据库加密
* HTTPS 数据传输

---

# 六、非对称加密（Asymmetric）

## 👉 本质

```text
公钥 + 私钥
```

---

## 👉 是否可逆

```text
✅ 可逆
```

---

## 👉 两种用法（重点🔥）

### 1️⃣ 加密

```text
公钥加密 → 私钥解密
```

---

### 2️⃣ 签名

```text
私钥签名 → 公钥验证
```

---

## 👉 常见算法

* RSA（最常见🔥）
* ECC（更先进）

---

## 👉 特点

✅ 安全（解决密钥分发）
❌ 慢

---

# 七、核心对比表（面试直接用🔥）

| 类型    | 可逆？ | 密钥     | 用途        |
| ----- | --- | ------ | --------- |
| 编码    | ✅   | 无      | 数据传输      |
| 哈希    | ❌   | 无      | 校验 / 密码   |
| 对称加密  | ✅   | 一个 key | 数据加密      |
| 非对称加密 | ✅   | 公钥+私钥  | 密钥交换 / 签名 |

---

# 八、关系总结（非常重要🔥）

```text
编码：只是换个表示方式（没有安全）

哈希：不可逆 → 用于校验和密码

加密：可逆 → 用于保护数据
  ├─ 对称：快（AES）
  └─ 非对称：安全（RSA）
```

---

# 九、真实系统怎么组合用（重点🔥）

## 👉 HTTPS

```text
RSA（非对称） → 传 AES key
AES（对称） → 加密数据
```

---

## 👉 登录系统

```text
密码 → bcrypt（哈希）
Token → JWT（签名）
```

---

## 👉 JWT

```text
HS256 → 对称
RS256 → 非对称
```

---

# 十、一句话记忆（超好用🔥）

```text
编码可逆但不安全
哈希不可逆用于校验
对称加密快但难分发
非对称加密安全但慢
```
