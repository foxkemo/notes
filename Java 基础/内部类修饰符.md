在 Java 里，**内部类的作用修饰符**主要指：👉“内部类可以用哪些访问控制修饰符，以及它们的作用范围”。

我帮你系统整理👇

---

# 🧠 一、内部类可以使用的修饰符

内部类本身可以用的访问修饰符只有 4 种：

|修饰符|是否可用|作用|
|---|---|---|
|public|✔|任意地方可见|
|protected|✔|包内 + 子类|
|default（无修饰）|✔|仅包内可见|
|private|✔|仅外部类内部可见|

---

# ⚙️ 二、四种修饰符详解

---

# 🟢 1️⃣ public 内部类

```java
class Outer {
    public class Inner {}
}
```

## ✔ 作用

👉 全部地方都能访问

```java
Outer.Inner inner = new Outer().new Inner();
```

---

## ✔ 特点

- 最开放
    
- 允许跨包访问
    

---

# 🟡 2️⃣ protected 内部类

```java
class Outer {
    protected class Inner {}
}
```

## ✔ 作用

👉

- 同包可访问
    
- 不同包只能子类访问
    

---

# 🔵 3️⃣ default（默认）

```java
class Outer {
    class Inner {}
}
```

## ✔ 作用

👉 仅同一个包内可访问

---

# 🔴 4️⃣ private（最重要🔥）

```java
class Outer {
    private class Inner {}
}
```

## ✔ 作用

👉 只能在 Outer 类内部使用

---

## ✔ 特点（重点🔥）

- 外部完全不可见
    
- 常用于封装实现细节
    

---

# 🧩 三、关键总结表

|修饰符|外部类访问|同包|子类|外部包|
|---|---|---|---|---|
|public|✔|✔|✔|✔|
|protected|✔|✔|✔|❌|
|default|✔|✔|❌|❌|
|private|✔|❌|❌|❌|

---

# ⚠️ 四、重要补充（很多人不知道🔥）

## ✔ 内部类 vs 外部类区别

外部类：

```java
public class A {}
```

👉 只能：

- public
    
- default
    

❌ 不能 private / protected

---

## ✔ 但内部类可以全部！

👉 因为内部类“属于外部类作用域内部”

---

# 🧠 五、为什么允许 private 内部类？

👉 因为：

> 内部类本来就是“外部类的实现细节”

例如：

```java
class Car {
    private class Engine {}
}
```

👉 Engine：

- 不需要外部知道
    
- 完全封装
    

---

# 🔥 六、使用建议（非常重要）

## ✔ 推荐用法

|场景|修饰符|
|---|---|
|工具/结构暴露|public static|
|内部实现|private|
|包内共享|default|
|扩展设计|protected|

---

# 🚀 七、一句话总结

👉 **内部类可以使用 public / protected / default / private 四种访问修饰符，用于控制其在外部类外的可见范围，private 是最常用于封装实现细节的方式**
