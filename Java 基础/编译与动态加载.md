# 一、核心问题拆解

你问的是：

> ❓ runtime 依赖为什么**不需要编译**，却可以**运行时使用**？

👉 本质是：

```text
Java 编译 ≠ Java 运行
```

---

# 二、Java 编译 vs 运行（本质区别）

## 1️⃣ 编译期（javac）

编译器只关心两件事：

```text
1. 类型是否存在
2. 方法签名是否正确
```

👉 如果代码里**没有直接引用某个类**：

```java
// 没有 import MySQLDriver
Class.forName("com.mysql.cj.jdbc.Driver");
```

✔ 编译器根本不检查这个类是否存在！

---

## 2️⃣ 运行期（JVM）

JVM 在运行时：

```text
需要类时 → 才加载类（懒加载）
```

👉 这就是：

## ⭐ 动态类加载（Dynamic Class Loading）

---

# 三、runtime 依赖为什么不需要编译

## ✔ 关键点：你没有“静态依赖它”

举个最经典例子👇

---

## 例子：JDBC 驱动

```java
Class.forName("com.mysql.cj.jdbc.Driver");
```

### 编译时：

- 只是一个字符串 `"com.mysql.cj.jdbc.Driver"`
    
- 编译器不会检查这个类
    
- ✔ 不需要 mysql-connector
    

---

### 运行时：

- JVM 执行：
    
    ```java
    Class.forName(...)
    ```
    
- 触发：
    

```text
ClassLoader 去 classpath 找类
```

✔ 找到了 → 加载成功  
❌ 找不到 → `ClassNotFoundException`

---

# 四、底层机制：类加载器（ClassLoader）

Java 的核心机制：

```text
Bootstrap ClassLoader
        ↓
Extension ClassLoader
        ↓
Application ClassLoader
        ↓
自定义 ClassLoader
```

---

## 动态加载流程

```text
Class.forName("xxx")
        ↓
当前 ClassLoader.loadClass("xxx")
        ↓
双亲委派模型
        ↓
在 classpath 中查找 .class 文件
        ↓
加载到 JVM（方法区）
```

---

# 五、为什么 Maven runtime 可以工作

## Maven 做了什么？

👉 Maven 在运行时：

```text
把 runtime 依赖加入 classpath
```

---

## JVM 做了什么？

```text
当代码执行到需要类时 → 才加载
```

---

## ✔ 关键组合

```text
Maven：把类放进 classpath
JVM：按需加载类
```

👉 所以 runtime 不需要编译阶段参与！

---

# 六、对比 compile vs runtime（本质）

|维度|compile|runtime|
|---|---|---|
|编译是否需要|✅|❌|
|代码是否直接引用|✅|❌（通常）|
|是否参与 classpath（运行）|✅|✅|
|加载方式|静态引用|动态加载|

---

# 七、两种依赖方式（非常重要）

## 1️⃣ 静态依赖（compile）

```java
import com.mysql.cj.jdbc.Driver;

Driver driver = new Driver();
```

👉 编译必须有这个类 ❗

---

## 2️⃣ 动态依赖（runtime）

```java
Class.forName("com.mysql.cj.jdbc.Driver");
```

👉 编译不需要  
👉 运行时才需要

---

# 八、再升一层：反射 + SPI

很多 runtime 依赖其实通过：

## 1️⃣ 反射

```java
Class<?> clazz = Class.forName("xxx");
```

---

## 2️⃣ SPI（Service Provider Interface）

例如 JDBC：

```text
META-INF/services/java.sql.Driver
```

JVM 自动加载实现类

---

# 九、为什么这样设计（核心意义）

## ✔ 解耦

```text
你的代码 ←→ 第三方实现
```

例如：

- JDBC 接口（compile）
    
- MySQL 驱动（runtime）
    

👉 可以随时替换数据库！

---

# 🔟 一句话总结

```text
runtime 不参与编译，是因为代码没有“静态依赖”它，
而是在运行时通过类加载机制（ClassLoader）动态加载。
```

---

# 🔥 再给你一个终极理解（很重要）

```text
Java 是“按需加载类”的语言（Lazy Loading）

不是所有类在启动时加载，
而是在第一次使用时加载。
```
