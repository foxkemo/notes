# 一、泛型方法：参数和返回值都可以指定吗？

## ✅ 结论

👉 **可以，而且这是最常见用法**

---

## ✅ 标准写法

```java
public <T> T method(T param) {
    return param;
}
```

👉 这里：

* 参数类型：`T`
* 返回值类型：`T`
* 调用时由编译器推断

---

## ✅ 示例

```java
public <T> T echo(T value) {
    return value;
}
```

调用：

```java
String s = echo("hello");  // T = String
Integer i = echo(123);     // T = Integer
```

---

## ✅ 更复杂：多个泛型

```java
public <K, V> V getValue(K key, V value) {
    return value;
}
```

---

## ✅ 参数和返回值“不同类型”

```java
public <T, R> R convert(T input) {
    return null;
}
```

👉 常见于：

* DTO 转换
* 工具类

---

## ✅ 结合约束

```java
public <T extends Number> T add(T a, T b) {
    return a; // 示例
}
```

---

## 🎯 核心理解

👉 泛型方法可以：

* 参数用泛型 ✔
* 返回值用泛型 ✔
* 两者不同 ✔
* 多个泛型 ✔

# 二、方法引用（Method Reference）

👉 本质：

👉 **Lambda 的简写**

---

# 三、常见 3 种（你说的“三种”）

---

# 1️⃣ 静态方法引用

## ✅ 语法

```java
类名::静态方法
```

---

## ✅ 示例

```java
Function<String, Integer> f = Integer::parseInt;
```

等价：

```java
Function<String, Integer> f = s -> Integer.parseInt(s);
```

---

# 2️⃣ 实例方法引用（对象::方法）

## ✅ 语法

```java
对象::实例方法
```

---

## ✅ 示例

```java
String str = "hello";
Supplier<Integer> s = str::length;
```

等价：

```java
Supplier<Integer> s = () -> str.length();
```

---

# 3️⃣ 类::实例方法（重要🔥）

## ✅ 语法

```java
类名::实例方法
```

---

## ✅ 示例

```java
Function<String, Integer> f = String::length;
```

等价：

```java
Function<String, Integer> f = s -> s.length();
```

---

👉 关键理解：

* 第一个参数 → 调用对象

---

# 四、补充（第4种：构造方法引用）

虽然你说三种，但这个也很常用👇

---

## 4️⃣ 构造方法引用

```java
类名::new
```

---

## ✅ 示例

```java
Supplier<User> s = User::new;
```

等价：

```java
Supplier<User> s = () -> new User();
```

---

# 五、总结表（非常重要）

| 类型    | 语法          | 示例                |
| ----- | ----------- | ----------------- |
| 静态方法  | 类::static方法 | Integer::parseInt |
| 对象方法  | 对象::方法      | str::length       |
| 类实例方法 | 类::实例方法     | String::length    |
| 构造方法  | 类::new      | User::new         |

---

# 六、方法引用的使用条件（核心）

👉 必须满足：

👉 **函数式接口的参数列表 和 方法签名匹配**

---

## 示例

```java
Function<String, Integer> f = String::length;
```

👉 因为：

```java
int length()
```

匹配：

```java
String -> Integer
```

---

# 七、结合你后端开发（重点）

你以后会经常看到：

```java
list.stream()
    .map(User::getName)
    .forEach(System.out::println);
```

# 八、一句话总结

👉

* 泛型方法：参数和返回值都可以用泛型 ✔
* 方法引用：Lambda 的简写，本质是“函数指针”
