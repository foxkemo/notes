# 一、什么叫“实例化后的泛型方法”？

泛型方法本身是：

```java
public <T> T echo(T value) {
    return value;
}
```

👉 **调用时才确定类型（实例化 T）**

---

## ✅ 1. 自动推断（最常用）

```java
String s = echo("hello");   // T = String
Integer i = echo(123);      // T = Integer
```

👉 编译器帮你完成：

```java
String s = this.<String>echo("hello");
```

---

## ✅ 2. 手动指定类型（很少用，但可以）

```java
String s = this.<String>echo("hello");
```

👉 这就是你说的：

👉 **“显式实例化泛型方法”**

---

# 二、实例化之后发生了什么？（核心）

👉 编译期：

```java
echo("hello") → echo<String>("hello")
```

👉 运行期（重要🔥）：

👉 **泛型被擦除（Type Erasure）**

等价于：

```java
Object echo(Object value) {
    return value;
}
```

---

# 三、关键本质（必须理解）

👉 Java 泛型是：

👉 **“编译期伪泛型”**

---

## 举个例子

```java
public <T> T test(T t) {
    return t;
}
```

---

👉 编译后大致变成：

```java
public Object test(Object t) {
    return t;
}
```

---

👉 为什么还能返回 String？

👉 因为编译器帮你加了**强转**

```java
String s = (String) test("hello");
```

---

# 四、带约束的泛型方法（更真实）

```java
public <T extends Number> T method(T t) {
    return t;
}
```

---

👉 擦除后：

```java
public Number method(Number t) {
    return t;
}
```

---

# 五、结合你熟的（Spring / MyBatis）

你见过：

```java
public <T> T getBean(Class<T> clazz)
```

---

## 调用：

```java
User user = context.getBean(User.class);
```

---

👉 本质：

* 传入 `Class<T>`
* 运行时靠这个类型做转换

👉 因为：

👉 **泛型在运行时已经没了，只能靠 Class 对象**

---

# 六、常见误区（很重要）

## ❌ 误区1：泛型方法运行时区分类型

```java
echo("a")
echo(1)
```

👉 ❌ 不是两个方法
👉 ✅ 只有一个方法（Object）

---

## ❌ 误区2：可以 new T()

```java
public <T> T create() {
    return new T(); // ❌ 错误
}
```

👉 原因：

* 运行时不知道 T 是谁

---

## ❌ 误区3：instanceof 泛型

```java
if (obj instanceof T) // ❌
```

---

# 七、什么时候“显式指定泛型”有用？

很少，但有👇

---

## 1️⃣ 推断失败时

```java
<T> void test(T t1, T t2)
```

```java
test(1, "a"); // ❌ 推断失败
```

👉 可以：

```java
this.<Object>test(1, "a");
```

---

## 2️⃣ 链式调用复杂时

```java
Collections.<String>emptyList();
```

---

# 八、一句话总结

👉
**泛型方法的“实例化”发生在编译期，运行时已经被擦除成本质类型（Object / 上界）**

---

# 九、给你一个面试级总结（建议背🔥）

👉

* 泛型方法在调用时确定类型
* 编译器进行类型推断或显式指定
* 运行时发生类型擦除
* 实际执行的是 Object 方法 + 强制类型转换
