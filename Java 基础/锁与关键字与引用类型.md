# 一、`synchronized` vs `Lock`（核心区别）

---

## 1️⃣ 本质区别（先记住）

> **synchronized 是 JVM 内置锁（关键字）**  
> **Lock 是 JUC 提供的显式锁（类）**

---

## 2️⃣ 使用方式对比

---

### 🔒 synchronized（代码块）

```java
synchronized (obj) {
    // 临界区
}
```

👉 自动：

- 加锁
    
- 释放锁（异常也会释放）
    

---

---

### 🔒 Lock（ReentrantLock）

```java
Lock lock = new ReentrantLock();

lock.lock();
try {
    // 临界区
} finally {
    lock.unlock();
}
```

👉 手动：

- lock()
    
- unlock()
    

---

## 3️⃣ 核心区别（面试重点🔥）

| 维度   | synchronized | Lock            |
| ---- | ------------ | --------------- |
| 实现   | JVM层         | JDK层            |
| 获取锁  | 自动           | 手动              |
| 释放锁  | 自动           | 必须手动            |
| 可中断  | ❌            | ✅               |
| 超时获取 | ❌            | ✅ (`tryLock`)   |
| 公平锁  | ❌            | ✅               |
| 条件变量 | ❌            | ✅ (`Condition`) |
| 性能   | JDK1.6后优化很好  | 更灵活             |
|      |              |                 |
|      |              |                 |
|      |              |                 |

---

## 4️⃣ 一个关键点（你必须知道）

---

### 🔥 synchronized 的底层

👉 基于：

- 对象头（Mark Word）
    
- Monitor（监视器锁）
    

---

👉 锁升级过程：

```text
无锁 → 偏向锁 → 轻量级锁 → 重量级锁
```

👉 优化：

- CAS
    
- 自旋锁
    

---

---

### 🔥 Lock 的底层

👉 基于：

> **AQS（AbstractQueuedSynchronizer）**

---

👉 核心：

- state（锁状态）
    
- FIFO 队列（线程等待队列）
    
- CAS 控制
    

---

# 二、什么时候用哪个？（实战🔥）

---

## ✅ 用 synchronized

👉 场景：

- 简单同步
    
- 代码块不复杂
    

👉 优点：

- 简单
    
- 不容易出错
    

---

## ✅ 用 Lock

👉 场景：

- 高并发
    
- 需要灵活控制
    

例如：

- tryLock（避免死锁）
    
- 多条件队列
    
- 公平锁
    

---

# 三、一个经典区别（很重要）

---

## ❓ 锁释放

### synchronized：

```java
synchronized (obj) {
    throw new RuntimeException();
}
```

👉 自动释放锁 ✅

---

### Lock：

```java
lock.lock();
throw new RuntimeException(); // ❌ 没释放锁
```

👉 必须：

```java
finally {
    lock.unlock();
}
```

# 四、Java 引用类型（JVM 内存核心🔥）

---

## 1️⃣ 四种引用类型

|类型|回收时机|说明|
|---|---|---|
|强引用|永不回收|普通对象|
|软引用|内存不足时|缓存|
|弱引用|GC就回收|ThreadLocal|
|虚引用|随时回收|跟踪对象|

---

---

## 2️⃣ 强引用（Strong Reference）

```java
Object obj = new Object();
```

👉 特点：

- 只要有引用 → 不会被回收
    

---

---

## 3️⃣ 软引用（SoftReference）

```java
SoftReference<Object> ref = new SoftReference<>(new Object());
```

👉 特点：

- 内存不够才回收
    

---

👉 用途：

- 缓存（图片缓存）
    

---

---

## 4️⃣ 弱引用（WeakReference）🔥

```java
WeakReference<Object> ref = new WeakReference<>(new Object());
```

👉 特点：

- 只要 GC → 立刻回收
    

---

👉 用途：

- ThreadLocalMap 的 key
    

---

---

## 5️⃣ 虚引用（PhantomReference）

```java
PhantomReference<Object> ref =
    new PhantomReference<>(obj, queue);
```

👉 特点：

- 不能获取对象
    
- 只用于监控回收
    

---

👉 用途：

- 内存回收通知
    
- 直接内存管理
    

---

# 五、一个关键问题（ThreadLocal 为什么会泄漏？）

---

👉 因为：

```text
ThreadLocalMap:
key → 弱引用
value → 强引用
```

---

👉 结果：

- key 被回收
    
- value 还在 ❌
    

👉 → 内存泄漏

---

# 六、引用类型总结（面试版🔥）

---

> 强引用最安全但不回收，软引用用于缓存，弱引用用于避免内存泄漏，虚引用用于跟踪回收过程。

---

# 七、把两部分串起来（高级理解🔥）

---

👉 你可以这样理解：

---

## 🔒 锁解决的是：

```text
多线程访问冲突问题
```

---

## 🧠 引用解决的是：

```text
对象生命周期管理问题
```

---

---

👉 在实际项目中：

- 锁 → 控制并发
    
- 引用 → 控制内存
    

---

# 八、一句话总结

---

> **synchronized 是简单可靠的内置锁，Lock 是灵活强大的并发工具；引用类型则决定对象什么时候被 GC 回收。**
