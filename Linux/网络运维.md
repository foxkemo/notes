给你一套**运维最常用的 Linux 网络排查命令清单（带使用场景）**，这基本就是面试 + 实战必备工具箱👇

---

# 🧰 一、连通性测试（最基础）

### 1. `ping` —— 测试主机是否可达

```bash
ping baidu.com
```

👉 看：

- 是否丢包
    
- 延迟（latency）
    

⚠️ 注意：很多服务器**禁 ping，不代表服务挂了**

---

### 2. `traceroute` —— 查看网络路径

```bash
traceroute baidu.com
```

👉 用来：

- 看请求经过哪些节点
    
- 卡在哪一跳（定位网络问题）
    

---

### 3. `mtr`（更高级 traceroute）

```bash
mtr baidu.com
```

👉 优点：

- 实时显示
    
- 丢包 + 延迟一起看
    

---

# 🌐 二、DNS 解析

### 4. `nslookup`

```bash
nslookup baidu.com
```

---

### 5. `dig`（更专业）

```bash
dig baidu.com
```

👉 常用：

```bash
dig +short baidu.com
```

---

# 🔌 三、端口连通性（比 ping 更重要！）

### 6. `telnet`

```bash
telnet baidu.com 80
```

👉 能连上说明：  
✔️ 端口是通的

---

### 7. `nc`（netcat，强烈推荐）

```bash
nc -zv baidu.com 80
```

👉 运维最常用：

- 检测端口是否开放
    
- 比 telnet 更现代
    

---

### 8. `curl`（HTTP 测试神器）

```bash
curl http://baidu.com
```

👉 常用：

```bash
curl -I http://baidu.com     # 只看响应头
curl -v http://baidu.com     # 查看详细过程
```

---

### 9. `wget`

```bash
wget http://baidu.com
```

👉 用来：

- 下载测试
    
- 验证 HTTP 服务
    

---

# 🔍 四、本机网络状态

### 10. `ip`（替代 ifconfig）

```bash
ip addr
ip route
```

---

### 11. `ss`（替代 netstat，重点！）

```bash
ss -tlnp
```

👉 看：

- 哪些端口在监听
    
- 哪个进程占用端口
    

---

### 12. `netstat`（老工具）

```bash
netstat -tlnp
```

---

# 🔥 五、流量 & 抓包分析

### 13. `tcpdump`（抓包神器）

```bash
tcpdump -i eth0 port 80
```

👉 用来：

- 分析请求有没有到达
    
- 网络问题终极武器
    

---

### 14. `iftop`

```bash
iftop
```

👉 看实时流量

---

### 15. `nload`

```bash
nload
```

👉 更直观流量显示

---

# 🧱 六、路由 / ARP / 链路

### 16. `arp`

```bash
arp -a
```

---

### 17. `route`

```bash
route -n
```

（已被 `ip route` 替代）

---

# 🚀 七、运维排障套路（非常重要）

当你遇到问题，按这个顺序查：

---

## ✅ 标准排查流程

### ① DNS 是否正常

```bash
dig 域名
```

---

### ② 网络是否可达

```bash
ping IP
```

---

### ③ 路径是否正常

```bash
traceroute IP
```

---

### ④ 端口是否开放

```bash
nc -zv IP 端口
```

---

### ⑤ 服务是否正常

```bash
curl http://IP:端口
```

---

### ⑥ 本机端口监听

```bash
ss -tlnp
```

---

### ⑦ 抓包确认

```bash
tcpdump
```

---

# 💡 一句话总结

👉 运维核心三板斧：

- **ping**（通不通）
    
- **nc / curl**（服务在不在）
    
- **tcpdump**（数据有没有到）
