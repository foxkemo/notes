MySQL 游标（Cursor）本质上是：**在存储过程里，一行一行处理查询结果的机制**。  
适合做“需要逐行逻辑处理”的场景（类似 Java 里的 `ResultSet.next()`）。

---

# 一、什么时候用游标？

👉 你可以先记一个原则：

> **能用 SQL 一次性解决 → 不要用游标**  
> **必须逐行处理（循环逻辑）→ 才用游标**

---

## ✅ 典型使用场景

- 逐行更新（复杂逻辑判断）
    
- 数据迁移 / 清洗
    
- 统计中需要“状态记忆”的逻辑
    

---

# 二、游标的完整使用流程（固定模板！）

MySQL 游标只能在**存储过程**中使用，流程是👇

### ✅ 5 步走

1. 声明游标
    
2. 声明结束标志
    
3. 打开游标
    
4. 循环读取
    
5. 关闭游标
    

---

# 三、标准模板（强烈建议背下来）

```sql
DELIMITER $$

CREATE PROCEDURE demo_cursor()
BEGIN
    -- 1️⃣ 定义变量
    DECLARE done INT DEFAULT 0;
    DECLARE v_id INT;
    DECLARE v_name VARCHAR(50);

    -- 2️⃣ 定义游标
    DECLARE cur CURSOR FOR 
        SELECT id, name FROM users;

    -- 3️⃣ 定义结束条件
    DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = 1;

    -- 4️⃣ 打开游标
    OPEN cur;

    read_loop: LOOP
        -- 5️⃣ 取数据
        FETCH cur INTO v_id, v_name;

        -- 6️⃣ 结束判断
        IF done THEN
            LEAVE read_loop;
        END IF;

        -- 7️⃣ 处理逻辑（重点）
        SELECT v_id, v_name;

    END LOOP;

    -- 8️⃣ 关闭游标
    CLOSE cur;
END $$

DELIMITER ;
```

---

# 四、核心语句拆解（非常重要）

## 1️⃣ DECLARE CURSOR

```sql
DECLARE cur CURSOR FOR SELECT ...;
```

👉 定义结果集

---

## 2️⃣ FETCH

```sql
FETCH cur INTO var1, var2;
```

👉 取一行数据

---

## 3️⃣ 结束标志（关键点 ⚠️）

```sql
DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = 1;
```

👉 没有数据时自动触发

---

## 4️⃣ LOOP + LEAVE

```sql
IF done THEN
    LEAVE loop_name;
END IF;
```

👉 类似 Java：

```java
while(rs.next()) { ... }
```

# 五、一个实际案例（面试级）

👉 场景：给所有用户余额 +100（但逐行判断）

```sql
DELIMITER $$

CREATE PROCEDURE update_balance()
BEGIN
    DECLARE done INT DEFAULT 0;
    DECLARE uid INT;

    DECLARE cur CURSOR FOR 
        SELECT id FROM users;

    DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = 1;

    OPEN cur;

    loop1: LOOP
        FETCH cur INTO uid;

        IF done THEN
            LEAVE loop1;
        END IF;

        UPDATE users 
        SET balance = balance + 100 
        WHERE id = uid;

    END LOOP;

    CLOSE cur;
END $$

DELIMITER ;
```

# 六、游标的缺点（很重要⚠️）

👉 一定要会说这段（面试加分）

### ❌ 性能差

- 一行一行执行（Row-by-Row）
    
- 无法利用 SQL 优化器
    

👉 被称为：

> **RBAR（Row By Agonizing Row）**

---

### ❌ 代码复杂

- 要写 handler
    
- 要写循环
    

---

# 七、替代方案（更推荐）

## ✅ 1. 用 UPDATE 直接做

```sql
UPDATE users 
SET balance = balance + 100;
```

---

## ✅ 2. 用窗口函数（你之前学过）

```sql
SELECT 
  id,
  ROW_NUMBER() OVER (ORDER BY id)
FROM users;
```

---

## ✅ 3. 用 JOIN / 子查询

👉 90% 情况可以替代游标

# 九、常见坑（必会⚠️）

### ❌ 忘记 handler → 死循环

---

### ❌ FETCH 顺序不匹配

```sql
FETCH cur INTO v_name, v_id; -- ❌ 顺序错
```

---

### ❌ 没 CLOSE → 资源泄露
