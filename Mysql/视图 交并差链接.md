# 🟢 一、视图（View）是什么

- 视图就是 **一个虚拟表**，本质上是 **一个 SELECT 查询**
    
- **不会存数据**（普通视图），查询时动态生成
    
- 语法示例：
    

```sql
CREATE VIEW student_view AS
SELECT uid, name, class_id
FROM student
WHERE status = 'active';
```

- 你对 `student_view` 的操作，实际上是操作 **这个 SELECT 的结果集**
    

✅ 关键点：**视图把复杂查询包装成“表”，所以理解交、并、表时，可以想象它们操作的都是“虚拟表”**

---

# 🟢 二、交、并、表的本质

从视图角度：

|操作|SQL 写法|理解|
|---|---|---|
|交集|`SELECT * FROM A INTERSECT SELECT * FROM B`|结果是两张表都包含的行 → 想象成两个视图“重叠部分”|
|并集|`SELECT * FROM A UNION [ALL] SELECT * FROM B`|两张表合并 → 想象成两个视图“堆叠在一起”|
|差集|`SELECT * FROM A EXCEPT SELECT * FROM B`|A 表有、B 表没有 → 想象成 A 视图去掉 B 的部分|

> ⚠️ 注意：
> 
> - UNION 默认去重，UNION ALL 不去重
>     
> - INTERSECT 和 EXCEPT 也默认去重（大部分数据库）
>

# 🟢 三、JOIN（表连接）理解

**视图角度思维**：

- 想象每张表都是一个视图
    
- JOIN = **两张视图拼接或配对行**
    
- 按照条件（ON / USING）生成结果
    

---

## 1️⃣ 内连接（INNER JOIN）

```sql
SELECT a.uid, a.name, b.score
FROM student a
INNER JOIN exam b ON a.uid = b.uid;
```

- 本质：
    
    1. “把 student 和 exam 当作两个视图”
        
    2. “挑出 uid 相同的行配对”
        
- 结果表 = **两个视图的交集 + 投影需要的列**
    

---

## 2️⃣ 外连接（LEFT / RIGHT / FULL JOIN）

|类型|含义|视图理解|
|---|---|---|
|LEFT JOIN|左表全部行，右表匹配的行|左视图全保留，右视图匹配行或 NULL|
|RIGHT JOIN|右表全部行，左表匹配|右视图全保留，左视图匹配行或 NULL|
|FULL JOIN|两表全部行|两个视图合并，匹配的行配对，不匹配补 NULL|

---

## 3️⃣ 半连接（SEMI JOIN）

- 常用 `EXISTS` 或 `IN` 实现
    
- 视图角度：
    
    - 只看左表
        
    - “左视图行是否存在于右视图”
        

```sql
SELECT *
FROM student s
WHERE EXISTS (SELECT 1 FROM exam e WHERE e.uid = s.uid);
```

- 想象：**左视图每行被右视图“打勾”或“过滤掉”**

# 🟢 四、组合操作（视图嵌套）

> 视图的强大之处在于，可以把复杂的交、并、表操作嵌套

```sql
CREATE VIEW active_scores AS
SELECT s.uid, s.name, e.score
FROM student s
JOIN exam e ON s.uid = e.uid
WHERE s.status = 'active';

SELECT uid, name
FROM active_scores
WHERE score > 90;
```

- 想象：`active_scores` = 一个虚拟表
    
- 外层 SELECT = 再次筛选这个视图
    

---

# 🟢 五、从视图角度理解几个常见误区

1. **视图不是物理表**
    
    - 每次查询都动态生成结果
        
    - 所以 JOIN / UNION 等操作都在查询时计算
        
2. **操作视图 = 操作虚拟表**
    
    - 可以加 WHERE、ORDER BY、聚合函数
        
    - 相当于把 SQL 当成“数据流图”
        
3. **多层嵌套视图**
    
    - 可以把复杂的交并表操作拆成“层级视图”，每层都是一个中间结果
        
    - 直观理解：每个视图就是“一个中间表格”，最终把它们组合成你需要的结果
