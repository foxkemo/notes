# 一、总原则（先记这个🔥）

👉

> **Redis 不存“空结构”，只要结构为空，key 就会被删除**

但：

> **String 是唯一例外（可以是空字符串）**

---

# 二、各数据结构是否可以为空

---

## 1️⃣ String（字符串）

✅ **可以为空**

```bash id="8d0k8g"
SET key ""
```

✔ key 存在
✔ value = 空字符串

---

👉 判断：

```bash id="z40a2n"
EXISTS key → 1
STRLEN key → 0
```

---

## 🔥 结论

> String 可以存“空值”

---

## 2️⃣ List（列表）

❌ **不可以为空**

```bash id="49p7lw"
RPUSH list a
LPOP list
```

👉 元素取完后：

```bash id="ht0m2g"
EXISTS list → 0
```

---

## 🔥 结论

> List 一旦为空 → key 自动删除

---

## 3️⃣ Set（集合）

❌ **不可以为空**

```bash id="m1h1ju"
SADD set a
SREM set a
```

👉 删除最后一个元素后：

```bash id="mejzj0"
EXISTS set → 0
```

---

## 4️⃣ Hash（哈希）

❌ **不可以为空**

```bash id="pq7uy5"
HSET user name zhang
HDEL user name
```

👉 删除最后一个 field 后：

```bash id="npttco"
EXISTS user → 0
```

---

## 5️⃣ ZSet（有序集合）

❌ **不可以为空**

```bash id="y9x36u"
ZADD zset 1 a
ZREM zset a
```

👉 删除后：

```bash id="k68yp4"
EXISTS zset → 0
```

---

## 6️⃣ Stream（流）

⚠️ **特殊：理论上可以“空”，但一般不会**

```bash id="5yhtvl"
XADD mystream * a 1
XDEL mystream id
```

👉 所有消息删掉后：

* key 可能还在（取决于版本/实现）
* 但通常生产环境不会这样用

---

# 三、总结表（直接背🔥）

| 数据结构   | 是否允许“空”存在 |
| ------ | --------- |
| String | ✅ 可以（""）  |
| List   | ❌ 不行      |
| Set    | ❌ 不行      |
| Hash   | ❌ 不行      |
| ZSet   | ❌ 不行      |
| Stream | ⚠️ 特殊情况   |

---

# 四、为什么这样设计？

👉 Redis 的设计原则：

```text
节省内存 + 简化判断
```

如果允许空结构：

* 会浪费 key 空间
* 增加判断复杂度

# 一、Redis 过期时间（TTL）是否只能用于 key？

## ✅ 结论

```text
Redis 的过期时间 只能作用在 key 上，不能作用在 value 的内部字段（field）
```

---

## 1️⃣ 为什么？

Redis 的数据结构是：

```text
key → value（整体）
```

👉 TTL 是绑定在 **key 上的元数据**

---

## 2️⃣ 示例

```bash
SET user:1 "zhangsan"
EXPIRE user:1 60
```

👉 60 秒后：

```text
整个 key 被删除
```

---

## ❗不能这样做（重点）

```text
Hash 中的某个 field 单独设置过期 ❌
```

例如：

```bash
HSET user:1 name zhangsan age 20
```

👉 ❌ 不能：

```text
只让 age 过期
```

---

## 3️⃣ 如果你想实现“field 过期”怎么办？

👉 常见方案：

### ✅ 方案1：拆 key（推荐🔥）

```bash
SET user:1:name zhangsan
SET user:1:age 20 EX 60
```

---

### ✅ 方案2：用 ZSet + 时间戳（高级）

👉 自己管理过期逻辑

---

### ✅ 方案3：JSON + 逻辑过期

👉 value 中加时间字段：

```json
{
  "age": 20,
  "expireAt": 1710000000
}
```

# 二、Redis 中 field 的值可以是 ""（空字符串）吗？

## ✅ 结论

```text
可以！Redis 支持空字符串 "" 作为 value 或 field 的值
```

---

## 1️⃣ 示例

```bash
HSET user:1 name ""
```

👉 是合法的

---

## 2️⃣ 区别（重点🔥）

### 情况对比：

| 情况         | 含义           |
| ---------- | ------------ |
| key 不存在    | 完全没有         |
| field 不存在  | hash 中没有这个字段 |
| field = "" | 字段存在，但值为空    |

---

## 3️⃣ 获取行为

```bash
HGET user:1 name
```

返回：

* `""` → 存在但为空
* `nil` → 不存在

---

## ❗Java 中的坑（你要注意🔥）

在 Java（比如用 Jedis / Redisson）：

```java
String val = redis.hget("user:1", "name");
```

👉 你必须区分：

```text
null   → field 不存在
""     → 存在但为空
```

---

# 三、延伸：哪些 Redis 数据结构可以“空”？（你之前问过）

## ✅ 可以为空值：

* String → `""`
* Hash → field = ""
* List → 可以空列表
* Set → 空集合
* ZSet → 空集合

---

## ❗但注意：

```text
Redis 不存“空 key”
```

👉 如果结构为空：

```text
key 会被删除
```

例如：

```bash
DEL 所有 field 后 → key 自动消失
```

---

# 四、总结（面试直接用🔥）

```text
1. Redis 的过期时间只能作用在 key 上，不能作用在 hash 的 field 上

2. 如果需要 field 级别过期，需要拆 key 或自己实现逻辑过期

3. Redis 支持空字符串 "" 作为 value

4. 要区分：
   null（不存在） vs ""（存在但为空）
```
