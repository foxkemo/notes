在 LeetCode 上刷算法题，并不需要掌握 Java 的全部特性（如 Spring、JVM 调优、多线程等），但需要对 **Java 集合框架（Collections Framework）**、**常用 API** 以及 **基本语法** 非常精通。

以下是为你整理的刷题必备 Java 知识清单：

---

### 一、 数组 (Array)
数组是最基础的数据结构。注意：Java 中数组长度是固定的。
*   **声明与初始化：** `int[] res = new int[n];` 或 `int[] res = {1, 2, 3};`
*   **获取长度：** `res.length` （注意是属性，不是方法）
*   **Arrays 工具类：**
    *   `Arrays.sort(nums)`：对数组进行升序排序。
    *   `Arrays.fill(nums, -1)`：快速填充数组。
    *   `Arrays.copyOf(nums, length)`：复制数组。
    *   `Arrays.toString(nums)`：打印数组（调试常用）。
    *   `Arrays.equals(arr1, arr2)`：比较两个数组内容是否相同。

---

### 二、 字符串 (String & StringBuilder)
字符串在算法题中出现频率极高。
*   **String (不可变字符串)：**
    *   `s.length()`：返回长度。
    *   `s.charAt(i)`：获取第 i 个字符（刷题最常用）。
    *   `s.toCharArray()`：转成字符数组 `char[]`（方便修改字符）。
    *   `s.substring(start, end)`：截取子串（左闭右开）。
    *   `s.equals(s2)`：**判断字符串相等必须用 equals**，不能用 `==`。
    *   `s.indexOf(char/str)`：查找字符下标。
    *   `String.valueOf(anyType)`：将其他类型转为字符串。
*   **StringBuilder (可变字符串)：**
    *   用于频繁修改字符串（如拼接），效率远高于 `String + String`。
    *   `sb.append(str)`：拼接。
    *   `sb.reverse()`：反转字符串（经典操作）。
    *   `sb.deleteCharAt(index)`：删除某个字符。
    *   `sb.setCharAt(index, char)`：修改。
    *   `sb.toString()`：最后转回 String。

---

### 三、 集合框架 (Collections) —— 核心中的核心
#### 1. List (列表)
*   **实现类：** `ArrayList<Integer> list = new ArrayList<>();`
*   `list.add(val)` / `list.add(index, val)`
*   `list.get(index)`
*   `list.size()`
*   `list.remove(list.size() - 1)`：删除最后一个（常用于回溯算法）。
*   `Collections.sort(list)`：对 List 排序。

#### 2. Map (映射)
*   **实现类：** `HashMap<Key, Value> map = new HashMap<>();`
*   `map.put(key, value)`
*   `map.get(key)`
*   **神技：** `map.getOrDefault(key, 0)` （如果 key 不存在则返回 0，统计次数神器）。
*   `map.containsKey(key)`：判断 key 是否存在。
*   `map.keySet()`：遍历所有 key。
*   `map.remove(key)`

#### 3. Set (集合)
*   **实现类：** `HashSet<Integer> set = new HashSet<>();`
*   `set.add(val)`：自动去重。
*   `set.contains(val)`：$O(1)$ 时间复杂度判断是否存在。

---

### 四、 栈与队列 (Stack, Queue, Deque)
刷 BFS、DFS、单调栈必备。
*   **Queue (队列)：** `Queue<Integer> q = new LinkedList<>();`
    *   `q.offer(val)`：入队。
    *   `q.poll()`：出队并返回。
    *   `q.peek()`：看队头。
*   **Deque (双端队列 - 推荐用作栈和队列)：** `Deque<Integer> dq = new ArrayDeque<>();`
    *   **当作栈用：** `dq.push(val)`, `dq.pop()`, `dq.peek()`。
    *   **当作队列用：** `dq.offerLast()`, `dq.pollFirst()`。
    *   *注意：Java 官方不再推荐使用 `Stack` 类，建议统一使用 `Deque`。*
*   **PriorityQueue (优先队列/堆)：**
    *   默认是**小顶堆**：`PriorityQueue<Integer> pq = new PriorityQueue<>();`
    *   设置**大顶堆**：`PriorityQueue<Integer> pq = new PriorityQueue<>((a, b) -> b - a);`
    *   常用于：Top K 问题。

### 五、 数学与数值处理
*   **最大最小值：** `Integer.MAX_VALUE` ($2^{31}-1$), `Integer.MIN_VALUE` ($-2^{31}$)。
*   **Math 类：**
    *   `Math.max(a, b)`, `Math.min(a, b)`
    *   `Math.abs(n)`：绝对值。
    *   `Math.pow(a, b)`：a 的 b 次方。
    *   `Math.sqrt(n)`：开平方。
*   **类型转换：**
    *   `Integer.parseInt("123")`：String 转 int。
    *   `(double) num`：强制类型转换。

---

### 六、 排序自定义比较器 (Comparator)
在处理二维数组或特殊排序规则时非常重要。
```java
// 对二维数组按第一列升序，如果第一列相同按第二列降序
int[][] intervals = {{1,3}, {2,6}, {8,10}};
Arrays.sort(intervals, (a, b) -> {
    if (a[0] != b[0]) return a[0] - b[0];
    else return b[1] - a[1];
});
```

---

### 七、 刷题时的重要细节（坑点）
1.  **对象判等：** `Objects.equals(a, b)` 或者 `s1.equals(s2)`，**绝对不要用 `==` 比较两个 Integer 对象或 String 对象的值**（除非是比较引用）。
2.  **溢出问题：** 如果题目说结果可能很大，记得用 `long`，或者在加法计算时先转为 long。
3.  **空指针：** 提交前习惯性检查 `if (root == null)` 或 `if (nums == null || nums.length == 0)`。
4.  **递归深度：** Java 的栈深度有限，如果深度超过 1000-2000 可能报 `StackOverflowError`。

### 学习建议：
如果你是初学者，建议先掌握 **ArrayList, HashMap, String, StringBuilder** 这四个 API，它们能解决 70% 的 Easy 和 Medium 题目。后续遇到二叉树、图、堆等题目时，再深入学习 **Deque** 和 **PriorityQueue**。
