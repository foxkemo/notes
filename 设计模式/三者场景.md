# 一、最终选择规则（核心🔥）

```text
入口/出口处理 → Filter
业务流程控制 → Handler
数据逐步加工 → Pipeline
```

---

# 二、最终落地场景总结（最重要）

## 1️⃣ Filter（最终用于“系统边界”）

👉 放在系统最外层

### ✔ 最终场景

```text
- 登录鉴权
- 权限校验
- 日志记录
- 请求限流
- 参数加解密
- 全局异常处理
```

---

### ✔ 特点（最终定位）

```text
不关心业务
统一入口/出口
所有请求都会经过
```

---

## ✔ 最终一句话

```text
Filter = 系统门口的“安检 + 包装器”
```

---

# 2️⃣ Handler（最终用于“业务决策链”）

👉 放在业务内部

---

### ✔ 最终场景

```text
- 订单处理流程
- 风控规则链
- 审批流
- 校验规则链
```

---

### ✔ 特点

```text
可以中断流程
可以修改业务数据
按顺序执行规则
```

---

## ✔ 最终一句话

```text
Handler = 业务流程的“决策流水线”
```

---

# 3️⃣ Pipeline（最终用于“数据处理链”）

👉 放在计算/转换层

---

### ✔ 最终场景

```text
- 数据清洗
- DTO 转换
- ETL 流程
- 图像/文本处理
- Netty 网络处理
```

---

### ✔ 特点

```text
一步一步加工数据
每一步输出新结果
不关心业务流程
```

---

## ✔ 最终一句话

```text
Pipeline = 数据加工工厂流水线
```

---

# 三、最终系统架构（真实项目🔥）

一个标准后端系统通常是这样：

```text
          Filter（系统入口）
                ↓
        Controller / Service
                ↓
        Handler（业务规则链）
                ↓
        Pipeline（数据处理）
```

---

# 四、一个完整订单例子（帮助你彻底理解）

---

## ✔ 1. Filter（门口）

```text
验证 token
记录日志
限流
```

---

## ✔ 2. Handler（业务）

```text
库存检查
优惠计算
风控判断
是否允许支付
```

---

## ✔ 3. Pipeline（计算）

```text
订单数据 → 计算折扣 → 计算税费 → 生成最终金额
```

---

# 五、最终决策口诀（面试直接背🔥）

```text
系统入口用 Filter，
业务流程用 Handler，
数据处理用 Pipeline。
```

---

# 六、最关键的“最终理解”（架构本质🔥）

```text
Filter   → 控制“请求是否进入系统”
Handler  → 控制“业务怎么执行”
Pipeline → 控制“数据如何变化”
```

---

# 七、一句话最终总结（最重要🔥）

```text
三者不是大小关系，而是三个不同层次：
Filter 管入口，Handler 管流程，Pipeline 管数据。
```
