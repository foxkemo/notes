很好，这一问就进入**“什么时候用哪种模式”**的核心了。我们把三者放到**真实开发场景**里讲清楚：

👉 **责任链 vs Pipeline vs Filter（双向链）**

---

# 一、先给你一个决策结论（最重要）

👉 **责任链：谁来处理（可中断）**  
👉 **Pipeline：每一步都处理（数据加工）**  
👉 **Filter：前后增强（包裹执行）**

---

# 二、三者典型应用场景

---

# 1️⃣ 责任链（Chain of Responsibility）

## ✔ 场景关键词

- 选择处理者
    
- 多条件匹配
    
- 可中断流程
    

---

## ✔ 典型业务场景

### （1）审批流程（最经典）

```text
员工 → 主管 → 经理 → 总监
```

👉 谁有权限谁处理，直接结束

---

### （2）异常处理（Spring）

```text
ExceptionResolver1 → Resolver2 → Resolver3
```

👉 谁能处理异常谁处理

---

### （3）权限校验（分级）

```text
登录校验 → 角色校验 → 权限校验
```

👉 某一步失败直接终止

---

## ✔ 为什么用责任链？

👉 因为你需要：

- ✔ 动态决定“谁处理”
    
- ✔ 可以中途结束
    
- ✔ 解耦 if-else
    

---

# 2️⃣ Pipeline（管道）

## ✔ 场景关键词

- 数据流处理
    
- 每一步都执行
    
- 不关心“谁处理”，而是“怎么加工”
    

---

## ✔ 典型业务场景

### （1）数据处理（非常常见）

```text
原始数据 → 清洗 → 转换 → 聚合 → 输出
```

---

### （2）日志处理系统

```text
日志 → 解析 → 过滤 → 入库
```

---

### （3）编译器 / SQL 处理

```text
SQL → 解析 → 优化 → 执行计划
```

---

### （4）Netty（重点）

```text
数据包 → 解码 → 业务处理 → 编码 → 返回
```

---

## ✔ 为什么用 Pipeline？

👉 因为你需要：

- ✔ 每一步都执行
    
- ✔ 强调数据变换
    
- ✔ 类似 Stream 流式处理
    

---

# 3️⃣ Filter（双向链）

## ✔ 场景关键词

- 前后增强
    
- 横切逻辑（通用功能）
    
- 请求/响应包裹
    

---

## ✔ 典型业务场景

### （1）日志（最经典）

```text
请求前记录 → Controller → 响应后记录
```

---

### （2）权限认证（通用拦截）

```text
token 校验 → 放行 → 返回后处理
```

---

### （3）性能统计

```text
开始时间 → 执行 → 结束时间
```

---

### （4）事务（AOP 本质）

```text
开启事务 → 执行方法 → 提交/回滚
```

---

### （5）Spring Security

```text
认证 → 授权 → 安全处理
```

---

## ✔ 为什么用 Filter？

👉 因为你需要：

- ✔ before + after
    
- ✔ 包裹整个流程
    
- ✔ 不侵入业务代码
    

---

# 三、三者核心对比（应用维度）

|维度|责任链|Pipeline|Filter|
|---|---|---|---|
|是否可中断|✅|❌（一般）|✅|
|是否全部执行|❌|✅|✅|
|是否有 after|❌|❌|✅|
|关注点|谁处理|怎么加工|怎么增强|
|典型方向|控制流|数据流|横切逻辑|

---

# 四、一个真实系统里的组合（非常重要）

在 Spring Boot 中：

```text
HTTP 请求
   ↓
Filter（日志 / 安全）
   ↓
Interceptor（权限 / 参数）
   ↓
Controller
   ↓
AOP（事务 / 日志）
```

👉 你会发现：

- 外层：Filter（双向增强）
    
- 中层：责任链（控制流程）
    
- 内层：AOP（更细粒度增强）
    

---

# 五、怎么选？（实战判断法）

## ✅ 问自己 3 个问题：

---

### ❓1：是不是“谁处理就结束”？

👉 YES → **责任链**

---

### ❓2：是不是“每一步都要处理数据”？

👉 YES → **Pipeline**

---

### ❓3：是不是“我要在前后都插逻辑”？

👉 YES → **Filter / AOP**

---

# 六、一个经典误区（很重要）

❗很多人会把：

```text
责任链 = Filter
```

👉 这是错的

正确是：

👉 Filter = **责任链 + 递归 + 双向增强**

---

# 七、终极总结（一句话记忆）

👉 **责任链：选人干活（可停）**  
👉 **Pipeline：流水线加工（全干）**  
👉 **Filter：前后包裹（增强）**
